QGS (Quote Generation System)

The project, "QGS (Quote Generation System)," aims to provide an efficient and user-friendly platform for users to generate insurance quotes. It addresses the problem of streamlining the insurance quote generation process, making it easier for both insured individuals and agents to obtain accurate quotes for various insurance policies.

Technology Stack

  • Java: The primary programming language for building the application logic.
  • SQL: Structured Query Language for managing and querying the database.
  • JDBC: A Java-based API for connecting and interacting with the relational database.
  • Custom Exceptions: Custom exception classes for handling specific error scenarios.
  • Properties File: Usage of properties files for database configuration (jdbc.properties).
  • Database Management System: A relational database system (e.g., MySQL, PostgreSQL) to store user and policy data.

Project Details

Custom Exceptions (QGSException)

QGSException is a custom exception class designed to handle project-specific exceptions and provide meaningful error messages to developers or users when exceptional situations occur.

Model Classes

In the QGS project, model classes represent data entities within the system, such as Accounts, Policy, PolicyDetails, PolicyQuestions, and UserRole.

DAO (Data Access Object) Layer

The DAO layer in the QGS project is responsible for interacting with the database, performing CRUD operations, and handling exceptions related to database access.

Service Layer

The service layer in the QGS project contains the QGSServiceImpl class, which serves as an intermediary between the client and the DAO layer, encapsulating the business logic.

Database Configuration

The database connection setup in the QGS project is managed by the JDBCUtility class, which handles the reading of properties from the jdbc.properties file and uses them to establish a connection to the database.

User Authentication

The user authentication process in the QGS project is managed by the verifyCredentials method in the QGSServiceImpl class, responsible for validating login credentials and handling exceptions.

Account Creation

The account creation process in the QGS project is managed by methods in the QGSServiceImpl class, involving input validation and interaction with the DAO layer.

User Profile Creation

The QGS project allows the creation of user profiles, which includes validating usernames, passwords, and role codes.

Policy Creation

The QGS project allows the creation of policies, utilizing model classes and coordinating with the DAO layer to ensure accurate data storage.

User Role Management

The QGS project manages user roles, ensuring username uniqueness, and enabling admin users to generate detailed policy reports.

Report Generation

The QGS project provides functionality for generating detailed policy reports for admin users, interacting with the DAO layer to retrieve and generate comprehensive reports.
